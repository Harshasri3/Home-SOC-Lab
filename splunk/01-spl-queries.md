# SPL Queries

This document contains the primary Splunk Processing Language (SPL) queries used throughout the Home SOC Lab. These queries were used to monitor endpoint activity, investigate events, and analyze Sysmon logs collected from the Windows endpoint.

---

# 1. Process Creation Monitoring

## Objective

Identify newly created processes on the endpoint.

## Why it matters

Process creation events help analysts identify suspicious applications, malware execution, command-line activity, and parent-child process relationships.

## SPL Query

```spl
source="WinEventLog:Microsoft-Windows-Sysmon/Operational"
EventCode=1
| table _time Image CommandLine ParentImage User
| sort -_time
```

## What to investigate

- Process Image
- Command Line
- Parent Process
- User
- Execution Time

---

# 2. PowerShell Activity

## Objective

Identify interactive PowerShell execution.

## Why it matters

PowerShell is commonly used by administrators, but it is also frequently abused by attackers for execution, persistence, and post-exploitation activities.

## SPL Query

```spl
source="WinEventLog:Microsoft-Windows-Sysmon/Operational"
EventCode=1
Image="*\\WindowsPowerShell\\v1.0\\powershell.exe"
| table _time Image CommandLine ParentImage User
| sort -_time
```

## What to investigate

- User
- Command Line
- Parent Process
- Execution Path

---

# 3. Network Connections

## Objective

Review outbound network connections initiated by endpoint processes.

## Why it matters

Network connection events help identify external communications, suspicious destinations, unusual ports, and possible command-and-control traffic.

## SPL Query

```spl
source="WinEventLog:Microsoft-Windows-Sysmon/Operational"
EventCode=3
| table _time Image DestinationIp DestinationHostname DestinationPort Protocol
| sort -_time
```

## What to investigate

- Process
- Destination IP
- Destination Hostname
- Destination Port
- Protocol

---

# 4. Network Activity Summary

## Objective

Identify which applications generate the most network traffic.

## Why it matters

This provides a quick overview of endpoint communication and highlights processes that warrant further investigation.

## SPL Query

```spl
source="WinEventLog:Microsoft-Windows-Sysmon/Operational"
EventCode=3
| stats count by Image DestinationPort Protocol
| sort -count
```

## What to investigate

- Frequently communicating processes
- Common destination ports
- Unexpected protocols

---

# 5. Process Statistics

## Objective

Identify the most frequently executed processes.

## Why it matters

A process that suddenly executes much more often than expected may indicate suspicious activity.

## SPL Query

```spl
source="WinEventLog:Microsoft-Windows-Sysmon/Operational"
EventCode=1
| stats count by Image
| sort -count
```

## What to investigate

- Frequently executed applications
- Rare processes
- Unexpected executables

---

# 6. Parent-Child Process Investigation

## Objective

Analyze parent-child process relationships.

## Why it matters

Attackers often launch processes from unusual parent processes to evade detection.

## SPL Query

```spl
source="WinEventLog:Microsoft-Windows-Sysmon/Operational"
EventCode=1
| table _time ParentImage Image CommandLine User
| sort -_time
```

## What to investigate

- Parent Process
- Child Process
- Command Line
- User

---

# 7. DNS Queries

## Objective

Review DNS requests generated by endpoint processes.

## Why it matters

DNS activity can reveal malware beaconing, suspicious domains, and command-and-control communications.

## SPL Query

```spl
source="WinEventLog:Microsoft-Windows-Sysmon/Operational"
EventCode=22
| table _time Image QueryName QueryStatus User
| sort -_time
```

## What to investigate

- Query Name
- Process
- Query Status
- User
